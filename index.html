Features

What your AI sees
when PM Agent is connected.

Every tool follows the Model Context Protocol. Your assistant picks the right one—you just ask naturally.

✎

Decision Tracking

Log ADRs with full rationale. Your AI queries, links, and enforces them across sessions.

⚠

Blocker Management

Track and resolve blockers. Your AI knows what’s stuck and suggests unblock strategies.

≡

Dependency Mapping

See how decisions, tasks, and code connect. Know the blast radius before making changes.

⌧

Scope Analysis

Check feature requests against sprint capacity. Your AI knows what fits and what’s risky.

⚙

Rules Engine

Define hard or soft PM rules. The engine catches violations before they reach production.

⌫

Codebase Scanning

Scan affected modules when a decision changes. PM context meets source code.

Setup

One config block,
instant project awareness.

No API keys, no accounts, no global installs. Add this to your MCP config and restart.

.mcp.json
// project-level config
{
  "mcpServers": {
    "pm-agent": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@gida-concept/pm-agent-mcp-server"]
    }
  }
}
1
Install

Node.js v18+. Nothing to install—npx handles the download.

2
Configure

Drop the JSON snippet into your AI tool’s MCP config file.

3
Restart

MCP configs load on startup. Restart your editor or CLI session.

4
Go

Ask naturally: “What’s blocking me?” — PM Agent handles the rest.
