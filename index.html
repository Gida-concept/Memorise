Features

What your AI sees
when PM Agent is connected.

Every tool follows the Model Context Protocol. Your assistant picks the right one—you just ask naturally.

Decision Tracking

Log ADRs with full rationale. Your AI queries, links, and enforces them across sessions.

Blocker Management

Track and resolve blockers. Your AI knows what's stuck and suggests unblock strategies.

Dependency Mapping

See how decisions, tasks, and code connect. Know the blast radius before making changes.

Scope Analysis

Check feature requests against sprint capacity. Your AI knows what fits and what's too risky.

Rules Engine

Define hard or soft PM rules. Your AI catches violations before they land in production.

Codebase Scanning

Scan affected modules when a decision changes. PM context meets source code.

Setup

One config block,
instant project awareness.

No API keys, no accounts, no global installs. Add this to your MCP config and restart.

// .mcp.json — project-level config for Claude Code
{
  "mcpServers": {
    "pm-agent": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@gida-concept/pm-agent-mcp-server"]
    }
  }
}
1
Install

Node.js v18+ required.
Nothing else to install—npx handles it.

2
Configure

Drop the JSON snippet into your AI tool's MCP config.

3
Restart

MCP config loads on startup. Restart your editor or CLI.

4
Go

Ask naturally: "What's blocking me?" — PM Agent handles the rest.

MCP Tools

Every tool your AI
needs to manage context.

17+ endpoints covering decisions, blockers, scope, rules, dependencies, and codebase scanning.

pm_get_context Project overview — blockers, decisions, tasks, scope
pm_log_decision Record an architectural or process decision
pm_add_blocker Log a new blocker with description and severity
pm_resolve_blocker Mark a blocker as resolved
pm_add_note Save an observation or insight
pm_scope_check Check feature fit against sprint capacity
pm_add_rule Add a PM rule with hard or soft enforcement
pm_scan_codebase Scan code affected by a decision or change
pm_analyze_dependencies Map dependency chains between tasks and code
pm_get_timeline Activity timeline for recent project changes
pm_get_rules List all active PM rules
pm_check_rules Check a proposed action against active rules
